# yyaml documentation

yyaml is a high-performance YAML parser written in C with zero-copy parsing paths and a thin C++ binding layer for ergonomic integration. This site outlines the architecture, APIs, and benchmarks so you can embed yyaml confidently.

## Architecture at a glance
- **C core**: streaming lexer, DOM builder, and emitter tuned for cache locality
- **Memory model**: arena allocators with configurable ownership to avoid churn
- **Error handling**: structured error codes plus optional string diagnostics
- **C++ bindings**: RAII wrappers over the C handles, iterator-friendly traversal, and exception/`std::error_code` options
- **Extensibility**: hooks for custom allocators and I/O abstractions

## Documentation pipeline
1. **Doxygen** extracts XML from `src/` + `include/` (C) and `bindings/cpp/` (C++).
2. **Doxybook2** turns XML into Docusaurus-ready Markdown in `docs/languages/<lang>/api/`.
3. **Prepare** copies landing pages, language pages, and benchmarks into `docs/build/docusaurus/`.
4. **Docusaurus build** emits the final static site to `docs/build/output-html/`.
